## 1. 프로젝트 개요

### 1.1 기본 정보

| 항목          | 내용                                         |
| ----------- | ------------------------------------------ |
| **프로젝트명**   | K-Market WMS (Warehouse Management System) |
| **버전**      | Ver 1.0                                    |
| **고객사**     | K-Market Vietnam                           |
| **대상 국가**   | 베트남 (하노이, 호치민)                             |
| **프로젝트 성격** | 베트남 최대 한국식품 유통기업 창고관리시스템                   |

### 1.2 비즈니스 배경

- **K-Market**: 베트남 최대 한국식품 유통기업
- **점포 수**: 하노이 65개점 + 호치민 35개점 = **총 100개점**
- **물류 특성**: 한국 → 베트남 수입 물류 + 다점포 배송

### 1.3 핵심 가치

| 가치                    | 설명                     |
| --------------------- | ---------------------- |
| **신선도 (Freshness)**   | 온도별 재고 관리, 유통기한 자동화    |
| **신속성 (Speed)**       | 최적화된 피킹 및 점포 배송 시스템    |
| **신뢰성 (Reliability)** | 정확한 재고 관리, 데이터 기반 의사결정 |

## 3. 기능 구성

### 3.2 메뉴 구조

```
K-Market WMS
│
├── 📊 대시보드

│
├── 📦 입고관리
│   ├── 입고예정등록 (ERP 연동, 다품목 선택, 보관유형별 입고번호 자동생성)
│   ├── 입고예정조회
│   ├── 입고확정 (예정 vs 실입고 비교, 정상/불량 구분, 온도대별 분리)
│   └── 입고실적조회 (일별/월별 집계, 통계 차트, Excel)
│
├── 🚚 출고관리
│   ├── 출고지시등록 (고객선택, 품목추가, 출고유형)
│   ├── 출고할당 (FIFO/FEFO 자동 할당, 유통기한 우선)
│   ├── 출고검수 (피킹수량 검증, 오출고 방지)
│   └── 출고실적조회
│
├── 📋 발주관리
│   ├── 점포발주등록 (수동/AI 자동발주(ADS), 권장수량 계산)
│   ├── 발주서관리 (조회/수정, 출고지시 자동생성)
│   ├── 수입발주등록 (구매요청→승인→발주→선적→통관→입고예정)
│   ├── 구매요청관리
│   ├── 공급처별 발주분할
│   └── 선적정보/인보이스 (ETD/ETA, BL번호, 컨테이너, 통관서류)
│
├── 📊 재고 & 로케이션
│   ├── 재고관리
│   │   ├── 재고현황 (센터/창고/구역별, 온도대별, 유통기한 알림)
│   │   ├── 단품별 재고이력현황
│   │   ├── 재고조정
│   │   └── 재고이동 (로케이션 간 이동, 온도대 검증)
│   ├── 로케이션관리
│   │   ├── 구역관리 (2D) - RACK(A~H) × ZONE(Z01~Z10) × 5x10 그리드
│   │   └── 로케이션설정
│   └── 재고조사
│       ├── 재고조사일정관리
│       └── 실사재고등록 (PDA/Excel/수동 입력, 차이 자동계산)
│
├── 📁 마스터관리
│   ├── 품목마스터 (카테고리, 온도대, 바코드, 다국어 품명)
│   ├── 카테고리관리 (3단계 계층, 트리뷰)
│   └── 거래처관리 (공급처/고객/점포, 결제조건, 신용등급)
│
├── 📱 PDA 작업
│   ├── PDA 메인
│   ├── 상품정보조회 (바코드 스캔)
│   ├── 상품입고검수 (입고예정서 스캔, 수량 검수)
│   ├── 입고적재 (품목/로케이션 스캔, 온도대 검증)
│   ├── 상품피킹 (2단계 스캔, 실시간 진행률, 햅틱 피드백)
│   ├── 로케이션이동 (출발/도착 로케이션 스캔)
```

## 6. 핵심 프로세스

### 6.1 4대 핵심 프로세스

```
1. 🌍 물류센터 발주 프로세스
   한국 공급처 → 구매요청 → 승인 → 발주 → 선적 → 통관 → 입고예정

2. 🏪 점포 발주 프로세스
   점포 발주 → AI 권장수량 → 승인 → 출고지시 → 피킹 → 배송

3. 📦 입고 프로세스
   입고예정 → 입고확정(검수) → 적재(로케이션 배치) → 재고 반영

4. 🚚 출고 프로세스
   출고지시 → 재고할당(FIFO/FEFO) → PDA 피킹 → 검수 → 배송
```

### 6.2 온도대 관리

| 온도대 | 코드 | 온도 범위 |
|--------|------|-----------|
| 상온 | ROOM | 15°C ~ 25°C |
| 냉장 | COLD | 0°C ~ 10°C |
| 냉동 | FROZEN | -18°C 이하 |
