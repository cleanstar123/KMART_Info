## 1.2 역할별 담당 업무

### PM (Project Manager)

| 업무 | 상세 내용 | 산출물 |
|------|----------|--------|
| 프로젝트 계획 수립 | WBS 작성, 일정 계획, 리소스 배분 | 프로젝트 계획서 |
| 이해관계자 관리 | 고객사 미팅, 요구사항 협의, 변경 관리 | 회의록, 변경요청서 |
| 리스크 관리 | 리스크 식별, 대응 계획 수립 | 리스크 관리대장 |
| 진척 관리 | 주간 보고, 이슈 트래킹 | 주간보고서 |

### PL (Project Leader) / 시스템 아키텍트

| 업무        | 상세 내용                    | 산출물       |
| --------- | ------------------------ | --------- |
| 현행 시스템 분석 | AS-IS 프로세스 분석, 데이터 흐름 분석 | AS-IS 분석서 |
| 아키텍처 설계   | 시스템 구조, 기술 스택 선정         | 아키텍처 설계서  |
| 표준 정의     | 코딩 표준, 명명 규칙, API 표준     | 개발 표준 가이드 |
| 기술 검토     | 기술 PoC, 성능 검증            | 기술 검토 보고서 |

### 업무 분석가 (BA)

| 업무 | 상세 내용 | 산출물 |
|------|----------|--------|
| 현장 조사 | 창고 방문, 작업자 인터뷰, 업무 흐름 관찰 | 현장 조사 보고서 |
| 요구사항 수집 | 기능/비기능 요구사항 도출 | 요구사항 정의서 |
| 프로세스 설계 | TO-BE 프로세스 정의, 업무 흐름도 | 업무 프로세스 정의서 |
| 화면 기획 | 화면 목록, 화면 정의서, 스토리보드 | 화면 설계서 |

### DBA / 데이터 설계자

| 업무 | 상세 내용 | 산출물 |
|------|----------|--------|
| 데이터 분석 | 현행 데이터 구조 분석, 데이터 흐름 분석 | 데이터 분석서 |
| 논리 모델링 | ERD 설계, 엔티티 정의, 관계 정의 | 논리 ERD |
| 물리 모델링 | 테이블 설계, 인덱스 설계, 파티션 설계 | 물리 ERD, DDL |
| 데이터 이행 설계 | 이행 대상 정의, 매핑 정의 | 데이터 이행 계획서 |

### UI/UX 디자이너

| 업무 | 상세 내용 | 산출물 |
|------|----------|--------|
| 사용자 분석 | 페르소나 정의, 사용자 시나리오 | 사용자 분석서 |
| UI 설계 | 와이어프레임, 화면 레이아웃 | 와이어프레임 |
| UX 설계 | 사용자 플로우, 인터랙션 정의 | UX 플로우 문서 |
| 디자인 시스템 | 컬러, 타이포그래피, 컴포넌트 정의 | 디자인 가이드 |

## 1.4 세부 작업 내용

### 1.4.1 현행 분석 (AS-IS Analysis)

```
┌─────────────────────────────────────────────────────────┐
│                     현행 분석 체크리스트                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  □ 창고 레이아웃 파악                                     │
│    ├── 창고 면적, 구역 구분                               │
│    ├── 랙/선반 배치도                                    │
│    ├── 온도대별 구역 (상온/냉장/냉동)                      │
│    └── 작업자 동선                                       │
│                                                         │
│  □ 업무 프로세스 분석                                     │
│    ├── 입고 프로세스 (발주 → 입고 → 검수 → 적치)           │
│    ├── 출고 프로세스 (주문 → 피킹 → 검품 → 출하)           │
│    ├── 재고 관리 (실사, 이동, 조정)                       │
│    └── 반품/교환 프로세스                                 │
│                                                         │
│  □ 현행 시스템 분석                                       │
│    ├── 사용 중인 시스템 목록                              │
│    ├── 시스템 간 인터페이스                               │
│    ├── 데이터 흐름                                       │
│    └── 현행 시스템 문제점                                 │
│                                                         │
│  □ 데이터 분석                                           │
│    ├── 품목 수 (SKU)                                    │
│    ├── 일평균 입출고 건수                                 │
│    ├── 재고 회전율                                       │
│    └── 피크 시간대 물동량                                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.4.2 요구사항 정의

**기능 요구사항 분류**

| 모듈         | 주요 기능                     | 우선순위 |
| ---------- | ------------------------- | :--: |
| **마스터 관리** | 품목, 창고, 로케이션, 거래처, 사용자 관리 |  필수  |
| **입고 관리**  | 입고예정, ASN, 검수, GR확정, 적치지시 |  필수  |
| **출고 관리**  | 주문접수, Wave생성, 피킹, 검품, 출하  |  필수  |
| **재고 관리**  | 재고조회, 이동, 조정, 실사, 마감      |  필수  |
| **PDA**    | 입고확인, 피킹, 적치, 재고실사        |  필수  |
| **대시보드**   | 실시간 현황, KPI, 알림           |  선택  |
| **리포트**    | 입출고 현황, 재고 현황, 생산성        |  선택  |
| **인터페이스**  | ERP, OMS, TMS 연동          |  선택  |

**비기능 요구사항**

| 구분      | 요구사항     | 목표치         |
| ------- | -------- | ----------- |
| **성능**  | 화면 응답 시간 | 3초 이내       |
| **성능**  | 동시 접속자 수 | 100명 이상     |
| **가용성** | 시스템 가동률  | 99.5% 이상    |
| **보안**  | 사용자 인증   | ID/PW + 2FA |
| **보안**  | 데이터 암호화  | AES-256     |
| **확장성** | 창고 추가    | 무중단 확장      |

  - wms-api-svc (ClusterIP)

## B. 식품 유통 WMS 특수 요구사항

### 온도대 관리

| 온도대 | 온도 범위 | 관리 포인트 |
|--------|----------|------------|
| 상온 (ROOM) | 15~25°C | 습도 관리 |
| 냉장 (COLD) | 0~10°C | 온도 모니터링, 콜드체인 |
| 냉동 (FROZEN) | -18°C 이하 | 해동 방지, 재동결 금지 |

### FEFO (유통기한 우선 출고)

```java
// 피킹 시 FEFO 로직
@Query("SELECT s FROM Stock s " +
       "WHERE s.itemCd = :itemCd " +
       "AND s.status = 'NORMAL' " +
       "AND s.qty > 0 " +
       "AND s.expireDate > CURRENT_DATE " +
       "ORDER BY s.expireDate ASC, s.inboundDate ASC")
List<Stock> findAvailableStockByFEFO(String itemCd);
```

### 유통기한 체크 포인트

| 시점 | 체크 내용 | 액션 |

| 재고 | 유통기한 D-7 | 알림 발송 |
| 재고 | 유통기한 D-3 | 긴급 알림 |
| 출고 | 유통기한 임박 | FEFO 우선 출고 |
