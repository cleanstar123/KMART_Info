> [!tip] 배운 점
> - **네이밍 규칙 정리**
>   - Controller: `sync+명사` (동기화), `get+명사+By+조건` (단건), `동사+명사` (액션) — 단독 축약(`sync()`, `complete()`) 금지
>   - Service: `get+명사+역할`, `동사+명사` — `ForPda` 접미사 금지
>   - Mapper: `select/insert/update/upsert+명사+By/For` — `ForPdaSync`·`Pda` 삽입 금지
>   - Dart datasource: `sync+명사` (동기화 GET), `fetch+명사+By+조건` (단건 GET), `post+명사` (POST)
> - 리네임은 **Mapper → Service → Controller 순** (상향식)으로 진행해야 컴파일 오류 없이 추적 가능
> - Java 인터페이스 rename 시 반드시 **XML mapper `id` 속성도 동시에** 변경해야 MyBatis 매핑 오류 방지
