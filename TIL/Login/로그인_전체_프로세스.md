# WMS PDA 앱 - 로그인 전체 프로세스

앱이 켜지는 순간부터 로그인 성공/실패까지의 전체 흐름을 정리한 문서.

---

## 전체 흐름 요약

```
앱 실행
  └─► DB 초기화 (SQLite 생성 + 테이블 생성 + 시드 데이터 삽입)
        └─► 로그인 화면 표시 (LoginScreen)
              └─► 사용자 입력 (ID / PW)
                    └─► 로그인 버튼 클릭
                          ├─► [온라인] WMS API 인증 → 성공 시 로컬 캐싱
                          └─► [오프라인] 로컬 SQLite 캐시 인증
                                ├─► 성공 → authSessionProvider 저장 → 메인 화면 이동
                                └─► 실패 → 에러 메시지 표시
```

---

## 1단계: 앱 실행 - DB 초기화

**관련 파일:** `core/database/app_database.dart`

앱이 처음 실행되면 SQLite DB(`wms_pda_app.db`)를 초기화한다.

```
AppDatabase.database (싱글톤 getter)
  └─► _database가 null이면 _initDatabase() 호출
        ├─► getDatabasesPath() 로 기기 내 DB 경로 확보
        └─► openDatabase() 호출
              ├─► [최초 실행] onCreate: _createTables() + _seedUsers()
              └─► [이후 실행] onOpen: _createTables() (IF NOT EXISTS라 기존 테이블은 건드리지 않음)
```

### 생성되는 주요 테이블

| 테이블명 | 역할 |
|---|---|
| `wms_user` | 로그인 계정 정보 (암호화된 개인정보 포함) |
| `wms_login_log` | 모든 로그인 시도 이력 |
| `wms_if_item` | 상품 마스터 |
| `wms_if_location` | 위치 마스터 |
| 기타 | 피킹, 스캔 로그, 아웃박스 동기화 등 |

### 시드 데이터 삽입 (`_seedUsers`)

- 앱 **최초 설치 시** (`onCreate`)에만 실행
- ERP `TA_USERM` 기준 초기 계정들을 미리 삽입
- **비밀번호는 평문이 아닌 SHA-256 해시로 저장**

```dart
// 예시
'auth_pwd': PasswordHasher.hash('12345')
// → 'auth_pwd': '5994471abb01112afcc18159f6cc74b4f511b99806da59b3caf5a9c173cacfc5'
```

- `conflictAlgorithm: ConflictAlgorithm.ignore` → 이미 계정이 있으면 덮어쓰지 않음

## 2단계: 로그인 화면 표시

**관련 파일:** `presentation/screen/login_screen.dart`, `presentation/provider/login_provider.dart`

### Provider 초기화

`LoginScreen`이 마운트되면 Riverpod이 `loginProvider`를 생성한다.

```dart
final loginProvider = StateNotifierProvider.autoDispose<LoginNotifier, LoginState>(
  (ref) => LoginNotifier(ref, AuthRepository(...))
);
```

- `autoDispose`: 로그인 화면을 벗어나면 Provider가 자동으로 메모리에서 해제됨
- 초기 상태: `LoginState.initial()` (모든 필드 빈값, 에러 없음, 로딩 아님)

### 초기 LoginState

```dart
LoginState(
  idInput: '',
  pwInput: '',
  isLoading: false,
  isPasswordVisible: false,
  errorMessage: null,
  isSuccess: false,
)
```

## 3단계: 사용자 입력

**관련 파일:** `login_screen.dart`, `login_provider.dart`, `login_state.dart`

사용자가 ID/PW를 입력할 때마다 `LoginNotifier`의 메서드가 호출된다.

```
사용자 ID 입력
  └─► LoginIdField.onChanged → notifier.updateId(value)
        └─► state = state.copyWith(idInput: value, clearError: true)
              └─► 기존 에러 메시지가 있으면 자동으로 null로 초기화

사용자 PW 입력
  └─► LoginPwField.onChanged → notifier.updatePw(value)
        └─► state = state.copyWith(pwInput: value, clearError: true)
```

### copyWith의 clearError 동작

```dart
// clearError: true 를 전달하면 errorMessage를 강제로 null로 초기화
errorMessage: clearError ? null : (errorMessage ?? this.errorMessage),
```

이유: `errorMessage`는 `String?` 타입이라 `copyWith(errorMessage: null)`로 전달해도
`null ?? this.errorMessage` 때문에 기존 값이 유지된다. 그래서 별도 플래그 필요.

## 5단계: AuthRepository 하이브리드 인증

**관련 파일:** `data/repository/auth_repository.dart`

`LoginNotifier`가 `AuthRepository.login(id, pw)`를 호출한다.

```
AuthRepository.login(id, pw)
  │
  ├─► [1단계] WMS API 서버 인증 시도 (AuthRemoteDatasource.login)
  │     │
  │     ├─► 성공
  │     │     ├─► AuthLocalDatasource.upsertUser(user, pw)  ← 로컬 캐싱
  │     │     ├─► LoginLogDatasource.insertLog(type='API', result='SUCCESS')
  │     │     └─► UserModel 반환
  │     │
  │     └─► NetworkException (연결 불가 / 타임아웃)
  │           │
  │           └─► [2단계] 로컬 SQLite 캐시 폴백 (AuthLocalDatasource.login)
  │                 │
  │                 ├─► 캐시 히트 (이전에 온라인 로그인한 계정)
  │                 │     ├─► LoginLogDatasource.insertLog(type='LOCAL', result='SUCCESS')
  │                 │     └─► UserModel 반환
  │                 │
  │                 └─► 캐시 미스 (한 번도 온라인 로그인 안 한 계정)
  │                       ├─► LoginLogDatasource.insertLog(type='LOCAL', result='FAIL')
  │                       └─► OfflineFallbackException throw
  │
  └─► AuthException / ServerException (서버가 인증 거부)
        ├─► LoginLogDatasource.insertLog(type='API', result='FAIL')
        └─► 즉시 rethrow (로컬 폴백 없음)
```

> **핵심 설계 원칙:**
> 서버가 "너 인증 안 돼" 라고 응답한 경우(`AuthException`)는 로컬 캐시로 폴백하지 않는다.
> 네트워크가 아예 안 될 때(`NetworkException`)만 로컬 폴백이 동작한다.

## 6단계 (온라인 경로): WMS API 인증

**관련 파일:** `data/datasource/auth_remote_datasource.dart`, `core/network/dio_client.dart`, `core/error/api_error_handler.dart`

### API 요청

```
POST /api/auth/login
Body: { "auth_cd": "thien-hcm", "auth_pwd": "12345" }
```

- `DioClient.instance`: 싱글톤 Dio 인스턴스 (baseUrl, timeout 설정 포함)
- 평문 비밀번호를 HTTPS로 전송 → 서버에서 비교

### 응답 처리

```dart
// 성공 시
UserModel.fromJson(response.data)  // JSON → UserModel 변환

// 실패 시 (DioException)
ApiErrorHandler.handle(e)  // DioException → AppException 변환
```

### ApiErrorHandler 변환 규칙

| DioException 타입 | 변환 결과 |
|---|---|
| connectionError / Timeout | `NetworkException` → 로컬 폴백으로 |
| HTTP 401 | `AuthException('아이디 또는 비밀번호가 올바르지 않습니다.')` |
| HTTP 403 | `AuthException('접근 권한이 없습니다.')` |
| HTTP 5xx 등 | `ServerException('서버 오류가 발생했습니다.')` |

## 7단계 (온라인 경로): 로컬 캐싱

**관련 파일:** `data/datasource/auth_local_datasource.dart`, `core/security/field_encryptor.dart`, `core/security/password_hasher.dart`

API 인증 성공 후 `upsertUser(user, plainPwd)`로 로컬 DB에 저장한다.

### 저장 시 암호화 규칙

| 필드 | 저장 방식 |
|---|---|
| `auth_pwd` | SHA-256 해시 (단방향, 복호화 불가) |
| `emp_addr`, `tel_no`, `cel_no`, `e_mail` | AES-256 CBC 암호화 (양방향) |
| `emp_no`, `emp_nm`, `auth_cd` 등 | 평문 |

### SHA-256 해싱 (`PasswordHasher`)

```dart
PasswordHasher.hash('12345')
// → 'sha256 해시 문자열'
// 로그인 검증 시: hash(입력값) == DB의 hash값 비교
```

### AES-256 암호화 (`FieldEncryptor`)

```dart
// 암호화
final iv = IV.fromSecureRandom(16);  // 매번 랜덤 IV 생성
final encrypted = _encrypter.encrypt(value, iv: iv);
// 저장 형식: "base64(IV):base64(암호문)"
// → IV를 함께 저장해야 복호화 가능

// 복호화
final parts = combined.split(':');   // IV와 암호문 분리
final iv = IV.fromBase64(parts[0]);
_encrypter.decrypt64(parts[1], iv: iv);
```

- 키: 32바이트 고정 문자열 (현재 하드코딩 → 운영 전 Android Keystore 교체 예정)
- CBC 모드 + 랜덤 IV: 같은 값을 암호화해도 매번 다른 암호문이 나옴 (패턴 노출 방지)

### `conflictAlgorithm: ConflictAlgorithm.replace`

- `emp_no`가 PK → 동일 계정으로 다시 로그인하면 기존 행을 덮어씀
- 비밀번호 변경 시 자동으로 새 해시값으로 갱신됨

## 8단계 (오프라인 경로): 로컬 캐시 인증

**관련 파일:** `data/datasource/auth_local_datasource.dart`

```dart
final rows = await db.query(
  'wms_user',
  where: 'auth_cd = ? AND auth_pwd = ? AND use_yn = ?',
  whereArgs: [authCd, PasswordHasher.hash(authPwd), 'Y'],
  limit: 1,
);
```

- 입력한 비밀번호를 SHA-256 해시로 변환 후 DB의 해시값과 비교
- `use_yn = 'Y'` 조건으로 비활성 계정 차단
- 조회 성공 시 `_toModel()`로 변환 (개인정보 필드 AES 복호화)

## 9단계: 로그인 결과 처리

**관련 파일:** `login_provider.dart`, `presentation/provider/auth_session_provider.dart`

### 성공 시

```dart
final user = await _repository.login(id, pw);  // UserModel 반환

// 전역 세션에 유저 정보 저장
_ref.read(authSessionProvider.notifier).state = user;
// authSessionProvider: StateProvider<UserModel?> (null이면 비로그인 상태)

// 상태 업데이트
state = state.copyWith(isLoading: false, isSuccess: true);
```

`LoginScreen`의 `ref.listen`이 `isSuccess` 변화를 감지해서 화면 이동:

```dart
ref.listen(loginProvider, (prev, next) {
  if (prev != null && !prev.isSuccess && next.isSuccess) {
    context.go('/');  // go_router로 메인 화면 이동
  }
});
```

### 실패 시

```dart
on Exception catch (e) {
  state = state.copyWith(
    isLoading: false,
    errorMessage: e.toString().replaceFirst('Exception: ', ''),
    // AppException.toString()이 message를 반환하므로 앞의 'Exception: ' 접두사 제거
  );
}
```

- `LoginErrorBanner`가 `errorMessage != null` 조건으로 표시됨

## 10단계: 로그인 이력 기록

**관련 파일:** `data/datasource/login_log_datasource.dart`

모든 로그인 시도(성공/실패)는 `wms_login_log` 테이블에 기록된다.

```dart
await db.insert('wms_login_log', {
  'log_id':        _uuid.v4(),      // UUID로 고유 식별
  'emp_no':        empNo,           // 실패 시 null 가능
  'auth_cd':       authCd,
  'login_type_cd': loginTypeCd,     // 'API' or 'LOCAL'
  'result_cd':     resultCd,        // 'SUCCESS' or 'FAIL'
  'fail_reason':   failReason,      // 실패 사유 (성공 시 null)
  'login_dtm':     DateTime.now().toIso8601String(),
});
```

| login_type_cd | result_cd | 상황 |
|---|---|---|
| API | SUCCESS | 온라인 로그인 성공 |
| API | FAIL | 서버 인증 거부 (401 등) |
| LOCAL | SUCCESS | 오프라인 캐시 로그인 성공 |
| LOCAL | FAIL | 오프라인 + 캐시 없음 |

## 전체 레이어 호출 관계

```
[UI Layer]
LoginScreen
  └─► LoginNotifier (loginProvider)

[State]
LoginState (불변 객체, copyWith로 갱신)

[Business Logic]
AuthRepository
  ├─► AuthRemoteDatasource  (WMS API)
  ├─► AuthLocalDatasource   (SQLite)
  └─► LoginLogDatasource    (SQLite 로그)

[Infrastructure]
DioClient         → HTTP 통신
AppDatabase       → SQLite 싱글톤
PasswordHasher    → SHA-256 해싱
FieldEncryptor    → AES-256 암복호화
ApiErrorHandler   → DioException → AppException 변환

[Global State]
authSessionProvider → 앱 전역 로그인 유저 정보 보관
```

## 예외 타입 정리

| 예외 클래스 | 발생 조건 | 처리 |
|---|---|---|
| `NetworkException` | 연결 불가 / 타임아웃 | 로컬 캐시 폴백 시도 |
| `AuthException` | 401 / 403 서버 응답 | 즉시 에러 표시 (폴백 없음) |
| `ServerException` | 5xx 등 서버 오류 | 즉시 에러 표시 |
| `OfflineFallbackException` | 오프라인 + 캐시 없음 | 에러 표시 (온라인 첫 로그인 필요 안내) |
