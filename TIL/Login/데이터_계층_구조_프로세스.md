## 2. app_database.dart — 테이블 구조 정의

### 역할

- 앱 실행 시 SQLite DB 초기화
- 9개 테이블을 `CREATE TABLE IF NOT EXISTS`로 안전하게 생성
- 최초 실행 시 시드 유저 데이터 삽입

### 콜백 구분

| 콜백 | 실행 시점 | 하는 일 |
|---|---|---|
| `onCreate` | DB 최초 생성 시 한 번 | `_createTables()` + `_seedUsers()` |
| `onOpen` | 앱 실행 시마다 | `_createTables()` — 누락 테이블 자동 보완 |
| `onUpgrade` | version 올라갈 때 | 현재 미사용 (IF NOT EXISTS로 대체) |

### IF NOT EXISTS 전략

```
앱 실행 → onOpen → _createTables() 호출
  ↓
9개 테이블 전부 IF NOT EXISTS 체크
  ├── 테이블 있으면 → 건너뜀 (기존 데이터 보존)
  └── 테이블 없으면 → 자동 생성
```

나중에 테이블이 추가되어도 `_createTables()`에만 추가하면 자동 처리됨

### 관리 테이블 목록

```
wms_user            — 유저 계정 (인증 기준)
wms_login_log       — 로그인 이력
wms_if_item         — 상품 마스터
wms_if_location     — 로케이션 마스터
wms_pick_task_h     — 피킹 지시 헤더
wms_pick_task_d     — 피킹 지시 상세
wms_pick_scan_log   — 피킹 스캔 로그
wms_sync_outbox     — 동기화 발신함
wms_sync_log        — 동기화 이력
```

## 3. user_model.dart — 데이터 그릇

### 핵심 원칙

**UserModel은 항상 평문** — 암호화/복호화는 datasource 계층에서만 처리

### 필드

```dart
final String empNo;      // 사원번호  ex) '00000'
final String empNm;      // 표시용 이름 ex) 'ADMIN'
final String authCd;     // 로그인 ID ex) 'root'  ← empNm과 다름 (혼동 주의)
final String cntrCd;     // 센터코드  ex) 'VIETNAM'
final int    menuLevel;  // 메뉴 권한 레벨

final String? empAddr;   // 개인정보 (null 가능)
final String? telNo;
final String? celNo;
final String? eMail;
// auth_pwd 없음 — 비밀번호 해시는 DB 비교에만 쓰이고 모델에 포함 안 됨
```

### 생성자 3개 역할

| 생성자 | 사용 시점 | 암호화 처리 |
|---|---|---|
| `UserModel({...})` | 직접 생성 (테스트 등) | 없음 |
| `UserModel.fromJson(json)` | API 응답 JSON → 모델 | 없음 (HTTPS 평문 수신) |
| `UserModel.fromMap(map)` | DB Row → 모델 | 없음 (호출 측에서 먼저 복호화) |
| `toMap()` | 모델 → DB 저장용 Map | 없음 (호출 측에서 암호화) |

## 4. auth_local_datasource.dart — 로컬 SQLite 조작

### `login()` — 오프라인 폴백 로그인

```
입력: authCd, authPwd (평문)
  ↓
PasswordHasher.hash(authPwd) → 해시값 생성
  ↓
SELECT * FROM wms_user
  WHERE auth_cd = ? AND auth_pwd = ? AND use_yn = 'Y'
  ↓
행 있음 → _toModel() → UserModel 반환
행 없음 → null 반환
```

### `upsertUser()` — API 로그인 성공 후 로컬 캐싱

```
API 응답 UserModel + 평문 비밀번호
  ↓
auth_pwd  → PasswordHasher.hash()         → 해시 저장
emp_addr  → FieldEncryptor.encryptNullable() → 암호화 저장
tel_no    → FieldEncryptor.encryptNullable() → 암호화 저장
cel_no    → FieldEncryptor.encryptNullable() → 암호화 저장
e_mail    → FieldEncryptor.encryptNullable() → 암호화 저장
  ↓
INSERT OR REPLACE INTO wms_user (conflictAlgorithm: replace)
```

### `syncUsers()` — 서버 유저 목록 전체 동기화

트랜잭션 안에서 두 단계 처리:

```
1단계: 서버 목록에 없는 로컬 유저 → use_yn = 'N' (로컬 로그인 차단)

2단계: 서버 유저 처리
  신규 유저 (로컬에 없음) → INSERT, auth_pwd = ''
    → 첫 온라인 로그인 성공 시 upsertUser()로 채워짐
  기존 유저 (로컬에 있음) → UPDATE (auth_pwd 컬럼은 건드리지 않음)
```

### `_toModel()` — DB Row → UserModel (private)

```
DB Row (암호화된 값)
  ↓
FieldEncryptor.decryptNullable() 각 개인정보 필드 복호화
  ↓
UserModel (평문) 반환
※ auth_pwd는 UserModel에 포함 안 됨
```

## 5. auth_remote_datasource.dart — WMS API 통신

### Dio 설정

```dart
final Dio _dio = DioClient.instance;
// DioClient.instance: baseUrl, connectTimeout(5초), receiveTimeout(10초) 공통 적용
// 모든 remote datasource에서 공유 → 설정 변경 시 dio_client.dart 한 곳만 수정
```

### `login()` — POST /api/auth/login

```
POST http://210.219.229.46:7000/api/auth/login
Body: { "auth_cd": "root", "auth_pwd": "admin52345234" }  ← 평문 (HTTPS 보호)
  ↓
성공(200) → UserModel.fromJson(response.data)
실패       → DioException → ApiErrorHandler.handle() → AppException throw
```

### `fetchUsers()` — GET /api/auth/users

```
GET http://210.219.229.46:7000/api/auth/users
  ↓
성공 → List<Map<String, dynamic>> 반환 (UserModel 변환 없음)
     → Repository → syncUsers()로 전달
실패 → AppException throw
```

## 6. login_log_datasource.dart — 로그인 이력 기록

### 특징

- INSERT 전용 (SELECT/UPDATE 없음)
- `_uuid.v4()` 로 log_id PRIMARY KEY 자동 생성
- 로그인 성공/실패 여부와 관계없이 **항상** 기록

### `insertLog()` 파라미터

```dart
required String authCd,       // 시도한 로그인 ID
required String loginTypeCd,  // 'API' or 'LOCAL'
required String resultCd,     // 'SUCCESS' or 'FAIL'
String? empNo,                // 성공 시만 채워짐
String? failReason,           // 실패 시 사유
```

### 저장 예시

```
성공(API):  log_id='uuid', emp_no='00000', auth_cd='root', login_type_cd='API',   result_cd='SUCCESS', fail_reason=null
실패(LOCAL): log_id='uuid', emp_no=null,    auth_cd='root', login_type_cd='LOCAL', result_cd='FAIL',    fail_reason='아이디 또는 비밀번호가 올바르지 않습니다.'
```

## 7. 전체 로그인 프로세스 흐름

```
사용자: ID/PW 입력 후 로그인 버튼
  ↓
LoginNotifier.login()                        [login_provider.dart]
  ↓
AuthRepository.login()                       [auth_repository.dart]
  ↓
┌─────────────────────────────────────────────────────────────┐
│  1단계: WMS API 시도                                         │
│  AuthRemoteDatasource.login()                               │
│    POST /api/auth/login { auth_cd, auth_pwd }               │
│                                                             │
│  성공(200)                                                   │
│    → UserModel.fromJson()                                   │
│    → upsertUser() — 로컬 캐시 저장 (암호화)                  │
│    → insertLog(type:'API', result:'SUCCESS')                │
│    → UserModel 반환                                         │
│                                                             │
│  NetworkException (연결 불가/타임아웃)                        │
│    → 2단계로 넘어감                                          │
│                                                             │
│  AuthException (401/403)                                    │
│    → insertLog(type:'API', result:'FAIL')                   │
│    → rethrow — 로컬 폴백 없이 즉시 실패 (보안)               │
└─────────────────────────────────────────────────────────────┘
  ↓ NetworkException인 경우
┌─────────────────────────────────────────────────────────────┐
│  2단계: 로컬 SQLite 폴백                                     │
│  AuthLocalDatasource.login()                                │
│    hash(authPwd)로 wms_user 조회                            │
│                                                             │
│  캐시 있음 (use_yn='Y', 해시 일치)                           │
│    → _toModel() — 개인정보 복호화                           │
│    → insertLog(type:'LOCAL', result:'SUCCESS')              │
│    → UserModel 반환                                         │
│                                                             │
│  캐시 없음                                                   │
│    → insertLog(type:'LOCAL', result:'FAIL')                 │
│    → OfflineFallbackException                               │
└─────────────────────────────────────────────────────────────┘
  ↓
LoginNotifier
  성공 → state: loggedIn, user: UserModel
  실패 → state: errorMessage 화면 표시
```

## 8. 암호화 처리 위치 정리

```
UserModel          — 항상 평문, 암호화 코드 없음

API 수신           — HTTPS 평문 수신, 처리 없음
DB 저장 (쓰기)     — upsertUser() / syncUsers() 에서 FieldEncryptor.encrypt()
DB 조회 (읽기)     — _toModel() 에서 FieldEncryptor.decrypt()

비밀번호 저장      — PasswordHasher.hash() → 단방향, 복호화 불가
비밀번호 비교      — SQL WHERE auth_pwd = hash(입력값)
```

## 9. 두 datasource 비교

| | auth_local_datasource | auth_remote_datasource |
|---|---|---|
| 데이터 출처 | SQLite (로컬 파일) | WMS API (HTTP) |
| 주요 도구 | sqflite | Dio |
| 비밀번호 처리 | SHA-256 해시 비교 | 평문 전송 (HTTPS) |
| 개인정보 처리 | AES-256 암호화/복호화 | 평문 수신 |
| 실패 처리 | null 반환 | AppException throw |
| 현재 상태 | 시드 데이터로 동작 중 | API 미개발 (연동 대기) |
