
### 1. 오프라인 동기화 충돌 해결                                                                                           
  
  "PDA A에서 오프라인 상태로 LOT-001 재고 10개를 피킹 완료했는데, 같은 시간에 Web에서 해당 재고를 재고조정으로 0개로 변경했습니다. PDA가 온라인 복귀 시 어떻게 처리해야 하나요?" 
  
  - Conflict Resolution 전략 (Last Write Wins? Server Wins? Manual Merge?)
  - 이미 물리적으로 피킹된 상품은 어떻게 처리?                                                                                       
  - 사용자에게 어떤 수준의 피드백을 제공해야 하나요?                                                      

### 2. 동시 피킹 경합 상황

  "2명의 작업자가 동시에 같은 출고건의 같은 품목을 피킹하려고 바코드를 스캔했습니다. 한 로케이션에 재고가 5개인데, 둘 다 5개씩 피킹하려 합니다. 이 Race Condition을 어떻게 방지하나요?"                                
  
  - 분산 락(Distributed Lock) 적용 범위는? (출고건? 품목? 로케이션?)
  - 락 획득 실패 시 UX는? (대기? 다른 품목 먼저?)                                                                                   
  - 락 타임아웃 설정 기준은?                                                                                                                                                    
###  3. 바코드 스캔 오인식 처리                                                                                             

  "훼손된 바코드를 스캔했을 때 유사한 다른 품목 코드로 인식되는 경우가 있습니다. 이런 오스캔을 시스템적으로 어떻게 방지하나요?"
  
  - 체크섬 검증만으로 충분한가요?                                                                                                          
  - 2중 확인(품목 스캔 → 로케이션 스캔) 외 추가 검증 방법은?                                                                 
  - 오스캔 패턴 학습/감지 로직이 필요한가요?                                                                                          

### 4. 배터리 고갈 시 데이터 보존                                                                                         

"피킹 작업 중 PDA 배터리가 갑자기 방전되었습니다. 진행 중이던 작업 데이터(10건 중 7건 완료)는 어떻게 보존하나요?"

  - 로컬 DB 저장 주기는? (매 스캔마다? 품목 완료마다?)                                                                          
  - 앱 재시작 시 작업 복구 UX는?                                                                                                            
  - 부분 완료 상태의 서버 동기화 전략은?                                                                                                 


### 5. 대량 피킹 시 메모리 관리                                                                                             

  "하나의 Wave에 500개 품목, 50개 로케이션을 순회해야 합니다. 피킹 리스트 전체를 메모리에 올리면 저사양 PDA에서 OOM이 발생합니다. 어떻게 설계해야 하나요?" 
  
  - Pagination vs Lazy Loading vs 구간별 다운로드?                                                                          
  - 피킹 동선 최적화 데이터는 서버에서 계산? PDA에서 계산?                                                                  
  - 메모리 임계치 모니터링 및 자동 정리 전략은?                                                                                      

### 6. 다중 디바이스 상태 동기화                                                                                          

"작업자가 PDA-A로 피킹 중 기기 고장으로 PDA-B로 교체했습니다. 진행 중이던 작업을 PDA-B에서 이어서 할 수 있어야 합니다. 세션/상태 관리를 어떻게 설계하나요?"                                                                       
                                                                                                                                                                                                                                    
  - 작업 상태를 서버에 실시간 동기화? 로컬 우선?                                                                                                                                                                                    
  - 기기 전환 시 인증/인가 처리는?                                                                                                                                                                                                  
  - 이전 기기에서 동기화되지 않은 데이터 처리는?
